Quick Facts

Navigations and Shortcuts

  • Use the ARROW KEYS to move around
  • Use the CONTROL KEY and the ARROW KEYS to rotate the camera
  • Using the Mouse wheel will raise or lower the camera
  • Pressing + or - will raise or lower the camera
  • Pressing SHIFT key when moving or rotating will slow the displacment
  • If you check the Edge scrolling, moving the mouse pointer near the edge will scroll
  • Press G will turn the grid on/off
  • Press CTRL-S to save
  • Press CTRL-SHIFT-S to take a picture
  • Press CTRL-O to open a region
  • Press CTRL-N to create a new region
  • Press ALT-F4 to quit

Global Tools

Global tools are those that affect the whole region
  • Equalize - smooth the whole region
  • Raise terran - you will be prompted for an amount to raise the whole region
  • Lower terran - you will be prompted for an amount to lower the whole region
  • Erode terran - this will run a hydrologic water erosion simulation an the whole region
  • Erode terran bis - this will run a hydrologic water erosion simulation an the whole region, with different parameters than the previous one
  • Import image - same as the CTRL-ALT-SHIFT-R in SC4. you will be prompted for the waterlevel and for the scaling factor. You can also import SC4M files
  • Export - this will export the whole elevation range (in opposition to a jpeg which can only handle 256 levels) with a 10cm resolution

Zone Tools

Zone tools are terraforming tools that you can control with your mouse. Only the region under the cursor is affected while clicking.
  • You can change the shape of the cursor (square or round)
  • You can change the radius of the cursor and the strength of the tool
  • You can limit the change in elevation of the tool by using the min Height and max Height sliders
  • Flatten - this will flatten the area under the cursor to the height of the red pixel
  • Draw - this will flatten the area under the cursor to the height specified in the text box.
    YOU MUST PRESS ENTER FOR A CHANGE TO TAKE EFFECT
  • Smooth - this will smooth the area under the cursor
  • Rough - this will exagerate altitude differences in the area under the cursor
  • Make hills - this will make hills :), no difference between a square and a rounded shape
  • Make mountains - this will make some mountains
  • Make steep hills - this will make some steep hills
  • Make valley - this will carve the landscape
  • Make valley(new) - this will carve the landscape with a different shape
  • Make harbor - this will flatten the land 0.5m above the seaLevel so you can easily plop waterfront when in SC4
  • Water erode - this will run the hydrologic simulation on the area under the cursor
  • Water erode enhanced - this will run the hydrologic simulation on the area under the cursor with different parameters
  • Talus erosion - this will run a termal erosion simulation on the area under the cursor

Modding textures

Here is a list of textures you can change to make SC4Terraform pleasant to you eyes.

  • Water.bmp, is the translucent texture used for water
  • Wired.bmp, is the texture used for showing grid, if you change it, leave a 2 pixel black border or the grid won't show
  • Detail.bmp, is the noise texture applied to the ground

If you are familiar with HLSL you can tweak the 3 .fx files. If you are familiar then you don't need information on what to do with them. If you don't know what to do with them, then you're obviously not familiar, so I'd advise you to not touch them.

Changing Colors

The default coloration is stored in a file called basicColors.ini in the program directory, which is loaded at run time. You can make your own coloration by copying the basicColors.ini to another file and editing it in a text editor such as NOTEPAD. In SC4Terraformer you can load the new coloration file by clicking the 'Change Colors' button and selecting the new coloration file.

The guide below explains the contents of the coloration file and how to edit it:

The file contains 3 sections:
    background
    water
    land

"background" is a single color and does not vary. You may set it to any color that pleases you. The DEFAULT is color = FF2F88 (recall that the HTML color codes are found from the reference web sites. Since the presentation is done in HTML you should LOOK UP the color code from one of the reference sites) For those who are interested, this last number is a HEXIDECIMAL TRIPLET representing the RGB VALUES for the color selected.

"water" and "land" both SET THE COLOR of SPECIFIC altitudes above or below water level.

Numbers IN BETWEEN these depths are INTERPOLATED (that is shaded in a smooth gradient) between the colors.

Looking at an entry, you can see this is done with number pairs such as "200 = 00084A"

This means that at 200 meters BELOW water level the color is 00084A. NOTE: Do not add a negative sign to the water.. it is implied to be below water. You will see that the default for water is only shaded from 0 to 200 meters below water.. all deeper water is the same color (00084A). WHY? because the color at 200 AND at 6000 are BOTH THE SAME, therefore interpolation gives the same color regardless of depth. You could change this of course if you wished.

"land" is done in the same way as "water" with pairs of numbers such as "100 = 00CE00". This means that at the height of 100 meters, the color will be "00CE00" (again recall, you have to look these up at a reference site as given above). Each added pair of numbers set the color at added heigths. The next color is "200 = 189C18". This means that at height 200, the color will be 189C18. From 100 to 200 colors are INTERPOLATED again smoothly across the range of "in between" colors.

By changing these "color points" you can affect exactly the color-to-height-mapping that is pleasing to you. This also allows you to modify the color to nicely represent your own personal terrain habits. If you have lots of altitude across your region, you can choose a full range of colors, but if your maps are rarely more than say 1000 meters across a region, you may choose to have much finer color ranges to see your variations more easily.

For help on choosing HTML color codes, go here.

Adding universal tools

With v.08c comes a way for you to create your own tools. These tools can be found in the Universal Tools section of the toolbar.

Tools are shapes defined by a spline (a curve that passes through predefined points).
The shape is revolved around the center of the tool (ie the red point)
The curve is defined from the center to the outer circumference, with a height modification to apply to the terrain
The curve is defined in the interval [0,1], where 0 represents the red point (center), and 1 means the outer radius of the tool
In the file tools.ini, you define a tool by creating a new section then you define some control points for where the curve should pass through 

[Hill]
0=1
0.7=0.9
1=0

Here we have defined a tool that will cause elevation changes at the distances 0 (0%), 0.7 (70%), and 1 (100%) from the center to the radius.
So at center (0%) it will raise the terrain 1m (1), at 70% of the radius it will raise the terrain .9m (0.9) and at 100% of the radius there will be no change (0). This tool will make a hill
To avoid perfect symmetry you can also add the following options:

  • perturbHeight = min,max   Specifies a random variation of a height of any point to avoid symmetry. The Min and Max values determine a range for the random values.
  • perturbRadius = min,max   Specifies a random variation of a point from the center to avoid symmetry. The Min and Max values determine a range for the random values.
  • scaleFactor = radius   Specifies that the bigger the tool, the greater the change in elevation


Wow, a wealth of information there. So let's go over it now to make sure we didn't miss anything
[Hill]-The name of the new tool
0=1-At 0% distance from radius, height will change by 1m
.7=.9-At 70% distance from radius, height will change by 0.9m. NOTE: Positive values are an increase in height, and negative values are a decrease in height.
1=0-At 100% distance from radius, height will change by 0m.
perturbHeight=Min,Max-Specifies Minimum and Maximum values for a random multiplier that changes height values to avoid perfect symmetry
perturbRadius=Min,Max-Specifies Minimum and Maximum values for a random multiplier that changes distance from radius values to avoid perfect symmetry
scaleFactor=radius-Specifies that the tool's intensity will vary with it's radius

As you might be able to tell, it'll take a couple of trial and errors to get this right. Just try something and see what it does. Experiment.

For your benefit, here is a full example, it creates a kind of V shape (hence the name Canyon) with a small amount of raise at the border to make a smooth hill at the top of the canyon
The strength will be a function of the radius, and the tool is perturbed in both height and radius

[Canyon]
0 = -2
0.5 = -1.9
0.8 = -0.9
0.85 = 0.1
0.9 = 0.15
0.95 = 0
1 = 0
perturbHeight = .75,1
perturbRadius = .85,1.15
scaleFactor = radius

FAQs

  • This sucks, everything is lagging as hell
      SC4Terraformer is quite CPU and GFX intensive
  • Ok this lag, how can I speed up a bit?
      Turn off water rendering, turn off shadows, turn on show city borders, unlock any cities (relock them before saving), move the camera so it looks down. In short, the less you see, the faster it will be
  • What is the SC4M file for?
      I hope that one day, we can exchange region with full elevation range using this file format.
  • How can I prevent the program to pop the readme everytime I run it?
      Rename the Readme.html to another name
  • Why is the draw tool drawing at the same height as harbor tool even if I put 1000m in the height?
      You should press Enter after entering the desired height. The draw height is reflected in the status bar
  • How do I change the background color and/or the terrain colors?
      Instructions can be found here.
  • Why do I see borders on the terrain in the region view in SC4 even if I uncheck borders?
      You have saved your region in the Terraformer while the 'show city border' option is checked. Try a resave with city borders off.
  • I have a big region, but I want to crop it to make a smaller one. Is it possible?
      You can export your region as 16bit greyscale png, then using your image editor, you can crop the interesting part, and reimport the new 16bit png in a new region.
      BTW, you can with the same technique arrange multiple regions.
  • What is the mean of parameters for the Import Dialog?
      They are parameters to convert from greyscale to SC4 units ( ie meters )
      • The scale factor is how much 1 unit in the greyscale image represent in SC4, you can enter a number or choose from the predefined one
  • So why I if change the parameters and load a 16bit png or a SC4M or a TER the region is still the same ?
      Thoses parameters are only used when you import a 8bit greyscale or RGB picture
  • Why SC4TF ask me for an HTML when I export to SC4M?
      This html ( the source code of it ) will be included into the SC4M, as the config.bmp and will be displayed on import.
